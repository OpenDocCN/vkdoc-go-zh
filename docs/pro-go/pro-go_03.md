# 2. 将 Go 置于上下文中

Go，通常被称为 *Golang*，是一种最初在谷歌（Google）开发的语言，目前已开始广泛使用。Go 语言在语法上类似于 C，但它拥有安全的指针、自动内存管理以及我所遇到过的最有用且编写得最好的标准库之一。

## 为什么要学习 Go？

Go 几乎可用于任何编程任务，但最适合服务器开发或系统开发。其广泛的标准库包括对大多数常见服务器端任务的支持，例如处理 HTTP 请求、访问 SQL 数据库以及渲染 HTML 模板。它拥有出色的线程支持，全面的反射系统使得为平台和框架编写灵活的 API 成为可能。

Go 附带了一套完整的开发工具，并且有很好的编辑器支持，这使得创建高质量的开发环境变得容易。

Go 是跨平台的，这意味着您可以例如在 Windows 上编写代码，然后在 Linux 服务器上部署。或者，正如我在本书中演示的那样，您可以将应用程序打包到 Docker 容器中，以便轻松部署到公共托管平台。



### “坑”在哪儿？

学习`Go`语言可能颇具挑战，因为它是一种“固执己见”的语言，这种特性有时会让使用者感到沮丧。它的这些“主见”既有真知灼见，也有令人烦恼之处。那些真知灼见让`Go`成为一次清新愉悦的体验，例如允许函数返回多个结果，这样就不需要用单个值同时表示成功和失败的结果。`Go`还有一些突出的特性，比如对线程的直观支持，这些特性值得许多其他语言借鉴。

那些令人烦恼的“主见”则会让编写`Go`代码感觉像在与编译器进行一场旷日持久的辩论，一种“编程界的‘还有一件事……’”式的争执。如果你的编码风格与`Go`设计者的“主见”不符，那么你很可能会看到一大堆编译器错误。如果你像我一样，已经写了很长时间的代码，并从众多语言中养成了根深蒂固的习惯，那么当编译器反复拒绝你那些在过去 30 年的所有其他主流编程语言中都能编译通过的表达式时，你会发明出一些新颖独特的咒骂词语。

此外，`Go`明显偏向于系统编程和服务器端开发。例如，虽然有提供 UI 开发支持的包，但这并非`Go`的强项，并且有更好的替代方案。

### 真的那么糟糕吗？

别被吓到。如果你正在从事系统编程或服务器开发项目，`Go`是一门优秀且值得学习的语言。`Go`拥有创新且高效的特性。一位经验丰富的`Go`开发者可以用令人惊讶的少量代码和精力构建出复杂的应用程序。

学习`Go`时要知道这需要付出努力。编写`Go`时要知道，当你和语言设计者意见相左时，他们的偏好胜出。

### 你需要了解什么？

这是一本为有经验的开发者编写的进阶书籍。本书不教授编程基础，你需要理解诸如`HTML`等相邻主题才能跟上所有示例。

## 本书结构是怎样的？

本书分为三个部分，每个部分涵盖一组相关主题。

### 第 1 部分：理解 Go 语言

在本书的第 1 部分，我将介绍`Go`开发工具和`Go`语言本身。我将描述内置数据类型，展示如何创建自定义类型，并涵盖诸如流程控制、错误处理和并发等特性。这些章节会包含`Go`标准库中的一些特性，这些特性要么是为了支持语言特性的解释，要么是执行与所描述的语言特性紧密相关的任务。

### 第 2 部分：使用 Go 标准库

在本书的第 2 部分，我将介绍庞大`Go`标准库中提供的、最实用的包。你将学习关于字符串格式化、数据读写、创建`HTTP`服务器和客户端、使用数据库以及利用强大的反射支持等特性。

### 第 3 部分：应用 Go

在本书的第 3 部分，我将使用`Go`创建一个自定义的 Web 应用框架，该框架是一个名为`SportsStore`的在线商店的基础。本书的这一部分展示了如何将`Go`及其标准库结合使用，来解决实际项目中出现的各类问题。本书第 1 部分和第 2 部分的示例一次只关注一个特性，而第 3 部分的目的是展示特性的组合使用。

## 本书未涵盖哪些内容？

本书并未涵盖`Go`标准库提供的所有包，如前所述，标准库非常庞大。此外，我还省略了一些`Go`语言特性，因为它们主流开发中不太常用。我在本书中描述的特性是大多数读者在大多数情况下会需要的。
