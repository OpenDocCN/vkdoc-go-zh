# 7. 安全

尽管互联网最初被设计为能够抵御敌对代理攻击的系统，但它后来发展成了一个相对可信赖实体间的合作环境。唉，那样的日子早已一去不复返了。垃圾邮件、拒绝服务（DoS）攻击、网络钓鱼尝试等事件表明，任何使用互联网的人都要自行承担风险。

应用程序必须构建成能够在对抗性环境下正确运行。“正确”不再仅仅意味着程序的功能方面正确，还意味着要确保所传输数据的隐私性和完整性、仅限合法用户访问以及其他安全问题。

这当然会使你的程序更加复杂。让应用程序安全涉及一些*困难*且*微妙*的计算问题。试图自行解决（例如编写自己的加密库）通常注定会失败。相反，你需要使用安全专家设计的库。

如果这让事情变得更难，为什么还要费心呢？几乎每天都有关于信用卡信息泄露、政府官员运行的私人服务器被黑客攻击，以及系统因拒绝服务攻击而瘫痪的报道。许多此类攻击都是通过网络应用程序中的编码错误实现的，例如缓冲区溢出、跨站脚本攻击和 `SQL` 注入。但大量错误可以追溯到糟糕的网络处理：明文传输密码、请求了安全凭证却不进行检查，以及盲目信任你所处的环境。例如，一位同事最近购买了一个家用物联网（IoT）设备。他使用 `Wireshark` 查看它在网络上的行为，发现该设备发送的 `RTMP` 消息中携带了认证令牌 `admin.admin`。这是一个简单的攻击向量，甚至无需破解密码！一家知名公司生产的无人机使用的加密存在已知缺陷，可以被其他无人机“劫持”。一种日益常见的窃取数据的方法是充当“恶意”无线接入点，假装是当地咖啡馆的合法接入点，但监控所有通过的数据，包括你的银行账户信息。这些都是“唾手可得”的目标。数据泄露的范围可以通过“世界上最大的数据泄露”网站（[`http://www.informationisbeautiful.net/visualizations/worlds-biggest-data-breaches-hacks/`](http://www.informationisbeautiful.net/visualizations/worlds-biggest-data-breaches-hacks/)）来了解。

本章介绍了 `Go` 提供的基本加密工具，你可以将它们构建到你的应用程序中。如果你不这样做，而你的公司损失了一百万美元——或者更糟的是，你的客户损失了一百万美元——那么责任就会落到你头上。
