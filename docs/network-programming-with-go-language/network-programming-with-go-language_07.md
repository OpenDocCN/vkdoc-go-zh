# 连接模型

为了让两台计算机进行通信，它们必须建立一个路径，以便在一次会话中至少发送一条消息。为此，有两种主要模型：

- 面向连接
- 无连接

## 面向连接

为会话建立单个连接。双向通信沿着该连接进行。当会话结束时，连接被断开。这类似于电话通话。TCP 就是一个例子。

## 无连接

在无连接系统中，消息是相互独立发送的。普通邮件可以作为类比。无连接的消息可能无法按顺序到达。消息之间互不影响。IP 协议就是一个例子。UDP 是 IP 之上的一种无连接协议，通常作为 TCP 的替代方案，因为它轻量得多。无连接也称为非连接或无状态。

面向连接的传输可以建立在无连接传输之上——例如，TCP over IP。无连接的传输也可以建立在面向连接的传输之上——例如，HTTP over TCP。通过面向连接传输协议发送的消息之间存在某种关联，例如，用于保持顺序的序列号。保持状态可以实现功能和优化；但它也伴随着存储和计算的成本。

这些模型可能存在变体。例如，一个会话可能强制消息到达，但不保证它们按发送顺序到达。然而，这两种是最常见的。

连接模型并非协议变化的唯一方式。一个经常被期望的特性是可靠性；这意味着协议拥有逻辑来修复某些类型的错误；例如，TCP 会重传丢失的数据包。人们通常认为面向连接的协议是可靠的；但这并不总是成立（例如，MPLS）。网络协议的其他特性可能包括消息边界管理、传递顺序、错误检查、流量控制等。这些特性可以存在于一个协议层，而不存在于另一层，这也是网络协议栈种类繁多的部分原因。

有时，这些特性会在整个协议栈中被重新设计。这种特性重构的一个例子是 HTTP/3。在 HTTP/2 中，可靠性是通过第 4 层的 TCP 提供的。在 HTTP/3 中，TCP 被不可靠的 UDP 取代。可靠性将由另一种名为 QUIC 的协议提供。虽然 QUIC 被视为与 TCP 或 UDP 类似的传输层，但它运行在 UDP 之上。如你所见，分层模型并非一门精确的科学。
