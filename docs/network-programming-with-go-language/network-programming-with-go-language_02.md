# 第一版前言

学习一门新的编程语言总是充满乐趣，尤其是当它最终成为一门主流语言时。在 2009 年 Go 语言发布之前，我曾在莫纳什大学教授一门硕士级别的网络编程课程。学习一门新语言时，设定一个目标总是好的；但这一次，我没有再编写一个葡萄酒管理程序，而是决定将我的讲义内容围绕 Go 语言展开，而不是我（当时）惯用的教学工具 Java。

这个实验效果很好：除了 Go 尚未能匹敌的 Java 库的丰富性之外，所有的编程示例都转移得非常出色，并且在许多情况下比原来的 Java 程序更加优雅。

这本书就是这一实验的成果。随着 Go 的演进以及如 HTTP/2 等新技术的出现，我一直对本书进行更新。但如果它读起来像一本教科书，嗯，那正是因为它本身就是。网络编程涉及大量的理论和实践概念，本书涵盖了其中一部分，以及使用 Go 构建系统的实践方法。

就语言流行度而言，Go 显然在上升。它在 TIOBE 指数中已经升至第 16 位，在 PYPL（编程语言流行度指数）中排名第 18 位，在 RedMonk 编程语言排名中位列第 15 位。它通常被评为增长最快的语言之一。

无论是核心语言及其库的开发者，还是独立项目的开发者，其社区都在不断壮大。我试图将本书的范围限定在标准库以及 Go 树的“子仓库”内。虽然这排除了许多无疑能使许多编程任务变得更简单的优秀项目，但将本书限制在官方 Go 库范围内提供了一个清晰的边界。

本书假定读者具备基本的 Go 知识。重点在于使用 Go 构建网络应用，而不是语言本身的基础知识。网络应用不同于命令行应用，也不同于具有图形用户界面的应用，等等。因此，第一章讨论了网络程序的架构方面。第二章概述了本书中使用的 Go 特性。第三章关于套接字的内容涵盖了所有 TCP/IP 系统基础的 Go 版本实现。第四章、第五章和第六章在网络编程书籍中较为不同寻常。它们涵盖了将要使用的数据表示形式、网络交互如何进行，以及对于文本，使用哪些语言格式等主题。接着在第七章，我们探讨日益重要的安全主题。在第八章，我们研究了最常用的应用层协议之一——HTTP。接下来的四章是关于 HTTP 相关主题以及 HTTP 之上常见的承载数据格式——HTML 和 XML。在第十三章，我们探讨了网络编程的另一种方法——远程过程调用。第十四章和第十五章则进一步讨论了使用 HTTP 的网络编程方面。
