# 验收因素

分布式系统的验收因素与独立系统类似。它们包括以下内容：

- **可靠性**（系统不会崩溃或损坏）
- **性能**（内部执行操作的速度）
- **响应性**（你感觉操作的速度有多快）
- **可扩展性**（我们能否通过增加更多实例来提高容量？）
- **容量**（我们能持续向软件请求多少工作）
- **安全性**（操作经过身份验证）

目前，用户常常对分布式系统容忍比独立系统更差的表现。“哦，网络很慢”似乎成了一个可接受的借口。嗯，这其实并不合理，开发人员不应形成这样的思维定势：认为自己能控制的因素可以产生可忽略的影响。

另一个比较点是，分布式系统通常设计为以更智能的方式进行扩展，而不是简单地将独立程序运行多次并想办法在它们之间拆分输入。
