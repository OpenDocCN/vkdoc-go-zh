# 链表

链表在软件开发中扮演着基础角色。它们按从第一个到最后一个的顺序保存一系列项目。在 6.11 节讨论的单向链表中，包含项目的每个节点都指向序列中的下一个节点。在 6.12 节讨论的双向链表中，每个节点都向前指向列表中的下一个节点，并向后指向列表中的上一个节点。这使得我们可以从头到尾或从尾到头遍历列表。

由于链表的线性结构，访问特定项目所需的时间比数组或切片更长。需要逐个节点遍历列表，直到找到目标项目。在需要通过位置索引快速直接访问项目的应用中，数组或切片更受青睐。

我们已经看到了链表的两个特例：`nodestack`和`nodequeue`。在`nodestack`中，信息从链表结构的最左侧节点插入和移除，假设元素从左侧插入，遵循 LIFO（后进先出）原则。在`nodequeue`中，信息按照 FIFO（先进先出）原则从左到右插入链表结构。

在链表中，信息可以插入到任何位置（开头、中间、结尾）。每次插入都会向列表中添加一个新节点，该节点在单向链表中链接到下一个节点，或在双向链表中链接到下一个节点和上一个节点。我们将展示每种链表的实现。

表征链表的 ADT 操作如下：

| `First()` – 返回列表中的第一个节点 |
| `Size()` – 返回列表中的节点数 |
| `Insert(i, item)` – 创建项目并将其插入列表的第 i 个节点 |
| `RemoveAt(i)` – 移除并返回列表第 i 个节点中的项目 |
| `Append(item)` – 创建项目并将其插入列表的最后一个节点 |
| `IndexOf(item)` – 返回列表中包含该项目的节点位置 |
| `Items()` – 返回列表中所有项目组成的切片 |
